## 4. 时间段解析分析

### 4.1 解析规则

时间段格式识别：
- `HHMM-HHMM` (如 `2300-600` → 23:00至次日06:00)
- `HMM-HMM` (如 `400-700` → 04:00至07:00)
- 支持 `~` 或 `至` 分隔符

跨天判断逻辑：
- 开始时间 >= 12 且 结束时间 < 12
- 开始时间 > 结束时间

### 4.2 解析成功率

| 状态 | 数量 | 占比 |
|------|------|------|
| 已解析 | 178 | 76.7% |
| 未解析 | 54 | 23.3% |

未解析原因分析：
- 纯文本描述（如"低温 -5 度"、"全场浇"）
- 不包含时间段格式
- 天气/事件描述类内容

## 6. 数据模型设计

### 6.1 核心模型：DemandRecord

```python
class DemandRecord(models.Model):
    # 基本信息
    date = models.DateField('需求日期')
    content = models.TextField('需求内容')
    original_text = models.TextField('原始文本')

    # 需求方信息
    demand_department = models.ForeignKey(DemandDepartment)
    demand_contact = models.CharField('联系人')

    # 区域信息
    zone = models.ForeignKey(Zone)
    zone_text = models.CharField('区域(文本)')
    is_global_event = models.BooleanField('全局事件')
    affected_zones = models.ManyToManyField(Zone)

    # 类别信息
    category = models.ForeignKey(DemandCategory)
    category_text = models.CharField('类别(文本)')

    # 时间段
    start_time = models.TimeField()
    end_time = models.TimeField()
    crosses_midnight = models.BooleanField()

    # 审批流程
    status = models.CharField()
    submitter = models.ForeignKey(Worker)
    approver = models.ForeignKey(Worker)
```

### 6.2 字典模型

**DemandCategory（19条）：**
- 全局事件类：停水停电、项目施工
- 工作类别类：项目配合、配合走场、Improvement、Learning等17种
- 区域需求类：通用类别

**DemandDepartment（4条）：**
- FES、FAM、ENT、其他
