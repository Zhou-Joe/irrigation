# Maxicom 数据同步系统 — 现场部署教程

> **目标**：把 Maxicom 灌溉控制器的运行数据，自动同步到 Django 服务器，供网页端查看。
>
> **适用场景**：Maxicom 软件 + 数据库在一台离线 Win7 电脑上（机器 A），Django 网站部署在旁边另一台 Windows + WSL 电脑上（机器 B）。

## 1. 整体架构（先看这个，了解原理）

### 两台电脑的分工

```
┌─────────────────────────────────┐       ┌─────────────────────────────────┐
│   机器 A（Maxicom 物联网机）       │       │   机器 B（Django 网站服务器）      │
│   系统：Windows 7                │       │   系统：Windows + WSL            │
│   特点：离线，不联网              │       │   特点：联网                     │
│                                  │       │                                 │
│   里面有什么：                    │       │   里面有什么：                   │
│   ① Maxicom 控制软件             │       │   ① Django 网站                  │
│   ② Maxicom2.mdb（数据库）       │       │      （运行在 WSL 里）           │
│   ③ MaxicomSync.exe（同步程序）   │       │   ② 用户通过浏览器访问           │
│                                  │       │                                 │
│   同步程序干两件事：              │       │                                 │
│   1. 读取本地 Maxicom2.mdb       │       │                                 │
│   2. 通过网线把数据传给机器 B ──────────────→ 接收数据，存入网站数据库        │
└─────────────────────────────────┘       └─────────────────────────────────┘
              │                                            │
              └──────────── 一根普通网线直连 ────────────────┘
```

### 为什么要用 USB 网卡？

机器 A 的网口全被 Maxicom 硬件占用了，没有空闲网口和机器 B 连接。所以我们需要 **USB 转网口适配器**（USB 网卡），插到机器 A 的 USB 口上，相当于给它"加了一个新网口"，专门用来连机器 B。**Maxicom 原来的线一根都不用动。**

### 为什么同步软件装机器 A？

- 机器 A 上有数据库文件（Maxicom2.mdb），同步软件在机器 A 上直接读取，速度最快、最稳定
- 机器 A 虽然离线，但同步软件不需要上网，只需要通过网线把数据发给旁边的机器 B 就行
- **不需要共享文件夹**——同步软件直接读取机器 A 本地的数据库文件

## 3. 第一步：接硬件（USB 网卡 + 网线）

### 3.1 把 USB 网卡插到机器 A

1. 找到机器 A 主机（那个大铁箱子）上的 **USB 口**（方形的接口）
2. 把 USB 网卡**插进去**
3. Windows 7 会自动识别，右下角可能弹出"正在安装设备驱动程序"，等它装完
4. 装好后，USB 网卡上应该有一个网线插孔（RJ45 接口）

```
机器 A 主机背面示意：
  ┌──────────────────────────────┐
  │  电源  ┃ ┃ ┃ ┃  ← 原来的网口（插着 Maxicom，不要动！）  │
  │        ┃ ┃ ┃ ┃                                │
  │        [USB][USB]  ← USB 口，找空闲的           │
  │           ↑                                     │
  │      USB 网卡插这里                              │
  └──────────────────────────────┘
```

> ⚠️ **重要**：机器 A 原来插着 Maxicom 硬件的网线**千万不要拔**！USB 网卡是额外加的，和原来的网口互不干扰。

### 3.2 用网线连接两台电脑

1. 网线的一头插到 **机器 A 的 USB 网卡**上
2. 网线的另一头插到 **机器 B**主机上的网线口

```
  机器 A                              机器 B
  ┌─────────┐                       ┌─────────┐
  │ USB网卡 │═══════ 网线 ══════════│ 网口    │
  │  (新加)  │                       │ (已有的) │
  └─────────┘                       └─────────┘
```

> 插进去听到"咔嗒"一声就是插好了。拔的时候要按住水晶头上的小弹片。

## 4. 第二步：设置 IP 地址（让两台电脑能互相找到）

这一步要给两台电脑的"直连网卡"各设一个固定的 IP 地址，就像给它们各取一个"门牌号"，这样它们才能互相通信。

### 4.1 机器 A（Win7）设置 IP：`192.168.137.1`

1. 点电脑左下角 **「开始」按钮** → **「控制面板」**
2. 找到并点 **「网络和 Internet」** → **「网络和共享中心」**
3. 点左边栏的 **「更改适配器设置」**
4. 你会看到几个网络连接图标。找到那个新出现的（通常叫"本地连接 2"或"以太网"，可能标着 USB 网卡的牌子）——**这个就是 USB 网卡，不要选原来连 Maxicom 的那个**
   - 如果分不清哪个是新的：拔掉 USB 网卡看哪个图标消失，再插上它就回来了，那个就是
5. 在那个新网卡图标上 **右键 → 属性**
6. 在弹出的列表里找到 **「Internet 协议版本 4 (TCP/IPv4)」**，**双击**它
7. 选择 **「使用下面的 IP 地址」**，填入：
   ```
   IP 地址：      192.168.137.1
   子网掩码：     255.255.255.0
   ```
8. 下面的 **默认网关** 和 **DNS 服务器** 全部**留空**（不要填！这条线只连两台电脑，不连互联网）
9. 点 **「确定」** → 再点 **「确定」** 关闭所有窗口

### 4.2 机器 B（Win10/11）设置 IP：`192.168.137.2`

1. 右键桌面右下角网络图标 → **「打开网络和 Internet 设置」**
   - 或者：开始 → 设置 → 网络和 Internet → 状态 → 更改适配器选项
2. 找到那张 **连着网线的网卡**（通常叫"以太网"，**不要选 Wi-Fi**，也**不要选 vEthernet (WSL)** 那个虚拟网卡）
   - 如果机器 B 有多张网卡，选那张刚插上网线的物理网卡
3. **右键 → 属性**
4. 双击 **「Internet 协议版本 4 (TCP/IPv4)」**
5. 选 **「使用下面的 IP 地址」**，填入：
   ```
   IP 地址：      192.168.137.2
   子网掩码：     255.255.255.0
   ```
6. 默认网关和 DNS **留空**
7. 点 **「确定」** → **「确定」**

> ⚠️ **机器 B 可能有多个网卡**（连公司网的、连 WSL 的、连机器 A 的）。**只改连机器 A 的那张**，其他网卡不要动。机器 B 上网走别的网卡，不受影响。

### IP 地址对照表（记一下）

| 机器 | IP 地址 | 说明 |
|------|---------|------|
| 机器 A（Maxicom 机） | `192.168.137.1` | 同步软件在这里，读本地数据库 |
| 机器 B（Django 服务器） | `192.168.137.2` | Django 网站在这里，接收数据 |

## 5. 第三步：验证网络连通（ping 测试）

设置完 IP，要确认两台电脑能"对话"。

### 在机器 A 上测试

1. 点 **「开始」** → 在搜索框输入 `cmd` → 回车，打开黑色命令行窗口
2. 输入以下命令后按回车：
   ```
   ping 192.168.137.2
   ```
3. 看结果：
   - ✅ 如果显示 `来自 192.168.137.2 的回复: 字节=32 时间<1ms TTL=128`（反复 4 次），**说明通了**，继续下一步
   - ❌ 如果显示 `请求超时` 或 `传输失败`，**说明没通**，检查：
     - 网线两头插紧了吗？
     - IP 地址填对了吗？（机器 A 是 .1，机器 B 是 .2）
     - 改的是正确的网卡吗？（别改错成 Maxicom 那张）
     - Windows 防火墙可能拦截了 ping，可以暂时关闭防火墙试试

### 在机器 B 上测试（反向确认）

机器 B 的 cmd（或 PowerShell）里：
```
ping 192.168.137.1
```
同样能看到回复就 OK。

> 两边 ping 都通，硬件和网络部分就全部搞定了！后面是软件配置。

## 6. 第四步：机器 B 配置 Django 服务器

机器 B 上 Django 跑在 WSL 里，需要做两件事让它能接收机器 A 发来的数据。

### 6.1 确认 Django 在 WSL 里正常运行

在机器 B 打开 WSL 终端，进入项目目录，启动 Django（监听所有网络接口）：

```bash
cd /mnt/c/Users/.../irrigation/internal_server

DEBUG=True SECRET_KEY=你的密钥 SYNC_API_KEY=dev-sync-key-change-in-production \
    python manage.py runserver 0.0.0.0:8000
```

> 关键：用 `0.0.0.0:8000`（不是 `127.0.0.1:8000`），这样 Django 才接受来自局域网的连接。看到 `Starting development server at http://0.0.0.0:8000/` 就是启动成功。

### 6.2 设置 WSL 端口转发（关键！）

WSL2 和 Windows 是两个独立的网络环境，机器 A 直接访问机器 B 的 Windows IP 是到不了 WSL 里的 Django 的。需要在**机器 B 的 Windows 上**（不是 WSL 里！）做一次端口转发。

在**机器 B 上**：右键开始菜单 → **「Windows PowerShell（管理员）」**，粘贴以下命令并回车：

```powershell
# 获取 WSL 的内部 IP
$wslIp = (wsl hostname -I).Trim().Split()[0]

# 设置端口转发：Windows 的 8000 端口 → WSL 的 8000 端口
netsh interface portproxy add v4tov4 listenport=8000 listenaddress=0.0.0.0 connectport=8000 connectaddress=$wslIp

# 防火墙放行 8000 端口
netsh advfirewall firewall add rule name="Django Sync 8000" dir=in action=allow protocol=TCP localport=8000
```

> 这三条命令的作用：让机器 A 发往 `192.168.137.2:8000` 的数据，自动转发到 WSL 里的 Django。

> ⚠️ **WSL 每次重启后 IP 会变**，这三条命令需要重新跑一遍。建议存成一个 `.bat` 或 `.ps1` 脚本，开机时双击运行。文件内容就是上面三行（第一行加 `#`，后面照抄）。

### 6.3 验证机器 B 能被访问

在机器 B 的浏览器里打开 `http://localhost:8000/`，能看到 Django 页面就说明服务在跑。

## 7. 第五步：机器 A 放置同步软件

### 7.1 创建文件夹

在机器 A 上建一个文件夹，比如 `C:\MaxicomSync\`：
1. 打开 **「计算机」（我的电脑）**
2. 打开 **C 盘**
3. 在空白处右键 → **「新建」** → **「文件夹」**
4. 把文件夹名字改成 `MaxicomSync`

### 7.2 拷贝文件

把以下**两个文件**从 U 盘/移动硬盘拷到 `C:\MaxicomSync\` 文件夹里：

```
C:\MaxicomSync\
├── MaxicomSync.exe        ← 同步程序（41MB 的那个 exe）
└── sync_config.json       ← 配置文件
```

> 就这两个文件。**拷完后确认它们在同一个文件夹里**（配置文件必须和 exe 放在一起）。

## 8. 第六步：找到 Maxicom 数据库文件路径

配置文件里需要填数据库文件的完整路径，先找到它。

### 8.1 搜索 Maxicom2.mdb

1. 在机器 A 上打开 **「计算机」**
2. 打开 **C 盘**（或 Maxicom 软件安装的盘）
3. 右上角搜索框里输入：`Maxicom2.mdb`
4. 等它搜完，会看到一个叫 `Maxicom2.mdb` 的文件
5. **右键**这个文件 → **「属性」**，看到"位置"那一行，就是它所在的文件夹

### 8.2 常见位置（如果搜索太慢，直接看这几个地方）

Maxicom 数据库通常在以下位置之一：

```
C:\Maxicom\Data\Maxicom2.mdb
C:\Program Files\Maxicom\Maxicom2.mdb
C:\Program Files (x86)\Maxicom\Maxicom2.mdb
C:\RainBird\Maxicom2.mdb
```

### 8.3 拼出完整路径

把文件夹路径和文件名拼起来，例如找到的位置是 `C:\Maxicom\Data`，文件叫 `Maxicom2.mdb`，那完整路径就是：

```
C:\Maxicom\Data\Maxicom2.mdb
```

**把这个路径记下来**，下一步要用。

## 9. 第七步：修改配置文件

### 9.1 打开配置文件

1. 进入 `C:\MaxicomSync\` 文件夹
2. 找到 `sync_config.json`，**右键 → 「打开方式」→ 选「记事本」**
3. 你会看到类似这样的内容：

```json
{
  "server_url": "http://192.168.137.2:8000",
  "api_key": "dev-sync-key-change-in-production",
  "mdb_path": "C:\\Maxicom\\Data\\Maxicom2.mdb",
  "mdb_password": "RLM6808",
  "sync_interval_minutes": 5,
  "overlap_buffer_minutes": 2,
  "days_window": 7
}
```

### 9.2 需要改的地方

**只有 `mdb_path` 这一行几乎一定要改**（其他一般不用动）。把第六步找到的数据库路径填进去。

| 字段 | 填什么 | 要改吗 |
|------|--------|--------|
| `server_url` | `http://192.168.137.2:8000` | ❌ 已填好（这是机器 B 的 IP） |
| `api_key` | `dev-sync-key-change-in-production` | ❌ 已填好（要和机器 B Django 一致，一般不用改） |
| `mdb_path` | 你找到的数据库完整路径 | ✅ **改成实际路径** |
| `mdb_password` | `RLM6808` | ❌ 已填好（Maxicom 默认密码） |
| 其他字段 | — | ❌ 不用改 |

### 9.3 改 mdb_path 的注意事项（重要！）

JSON 格式里，**路径的反斜杠 `\` 必须写两遍 `\\`**。

比如数据库实际路径是：
```
C:\RainBird\Maxicom2.mdb
```
在配置文件里要写成：
```json
"mdb_path": "C:\\RainBird\\Maxicom2.mdb",
```

⚠️ **反斜杠一定要写双份！** 单份会导致配置解析失败。

### 9.4 保存

1. 改完后，记事本菜单 **「文件」→「保存」**（或按 Ctrl+S）
2. 关闭记事本
3. **再打开一次确认**改对了（路径、反斜杠都对）

### 配置文件示例（假设数据库在 `C:\RainBird\`）

```json
{
  "server_url": "http://192.168.137.2:8000",
  "api_key": "dev-sync-key-change-in-production",
  "mdb_path": "C:\\RainBird\\Maxicom2.mdb",
  "mdb_password": "RLM6808",
  "sync_interval_minutes": 5,
  "overlap_buffer_minutes": 2,
  "days_window": 7
}
```

## 10. 第八步：验证同步（跑一次测试）

### 10.1 命令行测试（推荐先做这个，能看清错误信息）

1. 确保机器 B 的 Django 已经在跑（见第四步 6.1）
2. 在机器 A 上：**「开始」→ 搜索 `cmd` → 回车**，打开命令行
3. 输入以下命令进入软件目录（注意有空格）：
   ```
   cd C:\MaxicomSync
   ```
4. 运行测试同步：
   ```
   MaxicomSync.exe --once --days 7
   ```
5. 观察输出：

**✅ 成功的样子：**
```
MDB:     C:\RainBird\Maxicom2.mdb
Server:  http://192.168.137.2:8000
Window:  最近 7 天

  打开 MDB: Maxicom2.mdb
  [  3%] 正在打开数据库…
  [  5%] 读取 XA_WETHR …
  ...
  [ 60%] 上传 565 行到服务器…
========================================================
✓ 同步成功
```
看到 `✓ 同步成功` 就说明全部配置正确！🎉

**❌ 常见错误：**

| 错误信息 | 原因 | 解决 |
|---------|------|------|
| `✗ MDB 未找到: MDB file not found` | `mdb_path` 路径不对 | 检查路径、反斜杠是否写双份 |
| `✗ 连接失败: [WinError 10061]` | 连不上机器 B | 检查网线、IP 地址、Django 是否在跑、端口转发是否设置 |
| `✗ 服务器返回: 403` 或 `无效的同步密钥` | api_key 不一致 | 确认机器 A 配置文件的 `api_key` 和机器 B Django 启动时的 `SYNC_API_KEY` 完全一样 |
| `✗ 密码错误` | MDB 密码不对 | 确认 `mdb_password` 正确 |
| 中文显示成乱码 | 编码问题 | 在 cmd 里先执行 `chcp 65001` 再运行程序 |

### 10.2 测试通过后

测试成功就说明整个系统通了。接下来用图形界面日常运行就行。

## 11. 第九步：日常使用（日常怎么操作）

### 11.1 启动同步程序（每天开机后做一次）

1. 打开 `C:\MaxicomSync\` 文件夹
2. **双击 `MaxicomSync.exe`**
3. 弹出程序界面，上面有几个按钮：

```
┌──────────────────────────────────────────┐
│  Maxicom 数据同步                          │
│                                          │
│  数据窗口：[最近 7 天 ▼]  ← 可选 7/15/30 天 │
│                                          │
│  [立即同步]    ← 手动点一下，马上同步一次     │
│  [开启定时同步] ← 自动每 5 分钟同步一次      │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │ 日志区域（显示同步进度和结果）       │    │
│  │                                  │    │
│  └──────────────────────────────────┘    │

### 11.2 选择数据窗口

下拉框选要同步多久的数据：
- **最近 7 天**：最常用，数据量适中
- **最近 15 天**：同步半个月
- **最近 30 天**：同步一个月，数据量大，慢一些

### 11.3 手动同步一次

点 **「立即同步」**，看日志区域，出现"同步成功"就完成了。

### 11.4 开启自动同步（推荐）

点 **「开启定时同步」**，程序会每 5 分钟自动同步一次。可以最小化到系统托盘（右下角），后台自动运行。

### 11.5 日常注意事项

- ✅ 每天开机后双击启动程序，开启自动同步
- ✅ 确保机器 B 的 Django 一直在运行
- ⚠️ 如果机器 B 重启了（包括 WSL 重启），需要**重新跑一遍第四步 6.2 的端口转发命令**
- ⚠️ 程序运行时不要拔网线

## 12. 故障排查（出问题怎么办）

### 问题 1：双击 exe 没反应 / 闪退

**原因**：可能系统缺少依赖，或 Win7 版本太旧。

**解决**：
- 确认机器 A 是 **Win7 SP1**（Service Pack 1）。在「计算机」右键→属性里看。如果不是 SP1，先装 SP1 补丁
- 右键 exe → 属性 → 检查有没有"此文件来自其他计算机，已被锁定"的提示，有的话点「解除锁定」
- 试着用命令行运行（第五步），看具体报什么错

### 问题 1b：MDB 读取错误 -2147221005 "invalid class string"

**原因**：装错了版本的 EXE。Maxicom 软件是 32 位的，安装的数据库引擎也是 32 位的。如果你装的是 64 位 EXE，它找不到 32 位引擎。

**解决**：**必须使用 32 位版的 EXE**（`dist_win7_32bit\MaxicomSync.exe`，34MB），不要用 64 位版（`dist_win7\` 那个 41MB 的）。32 位 EXE 在 32 位和 64 位 Win7 上都能跑，且能找到 Maxicom 自带的引擎。

### 问题 1c：MDB 读取错误 -2147221164 "Class not registered"

**原因**：机器 A 上没有注册任何 DAO 数据库引擎，或者引擎损坏。

**解决**：
- 同步程序会自动尝试三种引擎版本（DAO.DBEngine.120 / .36 / .35），如果三个都不行才会报这个错
- 去 [微软官网](https://www.microsoft.com/en-us/download/details.aspx?id=13255) 下载安装 **Microsoft Access Database Engine 2010 Redistributable**（选 **AccessDatabaseEngine.exe** 即 32 位版，**不要**选 64 位的）
- 装完后重新运行同步程序

> ⚠️ 注意：必须装 **32 位** 的引擎（文件名 `AccessDatabaseEngine.exe`），不要装 64 位的（`AccessDatabaseEngine_X64.exe`），因为同步程序是 32 位的。

### 问题 2：同步失败，提示"连接失败"

**按顺序检查**：

1. **网线插好了吗？** 两头都按紧
2. **机器 A 能 ping 通机器 B 吗？** 在机器 A 的 cmd 里 `ping 192.168.137.2`
3. **机器 B 的 Django 在跑吗？** 在机器 B 浏览器打开 `http://localhost:8000/`
4. **端口转发设了吗？** 在机器 B 的 PowerShell（管理员）里执行 `netsh interface portproxy show v4tov4`，应该能看到 8000 端口的转发规则
5. **防火墙拦了吗？** 临时关闭机器 B 的防火墙再试

### 问题 3：同步失败，提示"403"或"无效的同步密钥"

**原因**：两边的密码（api_key）对不上。

**解决**：
- 机器 A 配置文件里的 `api_key` 值
- 机器 B 启动 Django 时 `SYNC_API_KEY=xxx` 的值
- 这两个必须**一模一样**（包括大小写）

### 问题 4：同步失败，提示"MDB 未找到"

**原因**：配置文件里的数据库路径不对。

**解决**：
- 用第六步的方法重新找 `Maxicom2.mdb` 的位置
- 确认路径写对了，**反斜杠写了双份** `\\`
- 在 cmd 里用 `dir 路径` 命令验证文件确实存在，比如 `dir C:\RainBird\Maxicom2.mdb`

### 问题 5：同步成功但网站上看不到新数据

**原因**：可能是 Django 那边的问题。

**解决**：
- 刷新网页
- 确认 Django 的数据库（不是 Maxicom 的）正常
- 看 Django 终端有没有报错

### 问题 6：中文显示成乱码（在 cmd 里）

**解决**：在 cmd 里先执行 `chcp 65001` 切换到 UTF-8 编码，再运行程序。

### 问题 7：WSL 重启后同步失效

**原因**：WSL 重启后 IP 变了，端口转发失效。

**解决**：在机器 B 重新跑第四步 6.2 的端口转发三条命令。

> 建议把端口转发命令存成 `转发.bat` 放桌面，WSL 重启后双击运行：
> ```powershell
> $wslIp = (wsl hostname -I).Trim().Split()[0]
> netsh interface portproxy add v4tov4 listenport=8000 listenaddress=0.0.0.0 connectport=8000 connectaddress=$wslIp
> ```
> （存成 `.ps1` 文件，右键"用 PowerShell 运行"）
